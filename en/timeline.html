Web Standards Implementation Timeline

Web sites, web based applications and device-based mobile applications need to comply with numerous Web Standards. Following is an outline of implementation dates, organized by year that the department must comply with on all public facing environments.

2017

December 2017: Web Renewal Initiative

  • Older content will be migrated to the future online archive sub-site. URLs and supporting web platforms are to be decommissioned.

2016

December 2016: Web Renewal Initiative

  • The transition plan targets migrating 100 per cent of the 1,500 existing websites. URLs and supporting web platforms are to be decommissioned.

April 01: Standard on Optimizing Websites and Applications for Mobile Devices

  • All Websites, web applications and device-based mobile applications must meet all the requirements for the Technical Specifications for the Web and Mobile Presence
  • Standard on Mobile Applications Project Overview

April 01: Standard on Web Usability

  • Phase 2: All Websites and Web applications and must conform to full standard
  • TBS standard full text requirements

March 31: Standard on Web Usability

  • New websites and Web applications must meet the requirements in Section 6 when they are published. Existing websites and Web applications must meet the requirements in Section 6 by March 31, 2016, with the exception of requirements 6.1.2 and 6.1.3, which must be met by July 31, 2013
  • TBS standard full text requirements

2015

December 2015: Web Renewal Initiative

  • The transition plan targets migrating 50 per cent of the 1,500 existing websites. URLs and supporting web platforms are to be decommissioned.

June 30: Standard on Web Interoperability

  • Phase III:
    • Implementation of HTML data syntax: RDFa 1.1 lite or later
    • Implementation of HTML data vocabulary: schema.org
  • HTML Data requirements are part of Appendix E

March: Government of Canada IPv6 Adoption Strategy

  • Deployment Phase: The second phase will focus on the IPv6 enablement of the principal GC externally-facing websites and is expected to be completed by the end of March 2015. Actions planned for this phase include:
    • Enabling principal-existing GC Internet-facing websites to be accessible by IPv6 users
    • Requiring all new Internet-facing websites and applications put in place starting April 2015 to be IPv6-enabled
    • Providing public servants transparent access to the public IPv6 Internet
  • Government of Canada IPv6 Adoption Strategy Overview

2014

December 31: Standard on Web Interoperability

  • Phase II: Implementation of character encoding UTF-8 and mark-up language HTML5
  • Character encoding requirements are part of Appendix C
  • Mark-up language requirements are part of Appendix D

December 2014: Web Renewal Initiative

  • The transition plan targets migrating 30 per cent of the 1,500 existing websites. URLs and supporting web platforms are to be decommissioned.

July 01: Standard on Optimizing Websites and Applications for Mobile Devices

  • Outsourcing the design, development and maintenance of device-based mobile applications to the departments identified as Government of Canada Centres of Expertise on Mobile or external service providers, with the exception of device-based mobile applications with secure logins.
  • Standard on Mobile Applications Project Overview

January 01: Standard on Optimizing Websites and Applications for Mobile Devices

  • Device-based mobile applications that are launched on or after September 1, 2013 need to comply with Appendix F
  • Standard on Mobile Applications Project Overview

Web Renewal Initiative

  • Outsourced web services, including web publishing, web analytics and online citizen engagement, will be procured.
  • The Principal Publisher will become operational.
  • A number of Centres of Expertise will be established.
  • All websites within the scope of Web Renewal will adopt the common GC web Search solution.

2013

December 18: Web Renewal Initiative

  • Canada.ca launches.

October: Consolidation of the following Web standards

  • Standard on Web Usability
  • Standard on Web Interoperability
  • Standard on Optimizing Web sites and Applications for Mobile Devices

October: Web Renewal Initiative

  • Web Renewal Project approved and Project Office established and staffed.

September 01: Standard on Optimizing Websites and Applications for Mobile Devices

  • Websites and web applications that are launched on or after September 1, 2013 will need to meet the requirements for the Technical Specifications for the Web and Mobile Presence.
  • Standard on Mobile Applications Project Overview

September: Government of Canada IPv6 Adoption Strategy

  • Enabling Phase: The goal is to enable federal organizations to develop their individual plans for the adoption of IPv6. Actions planned for this phase include:
    • Developing IPv6 architecture standards and technical requirements
    • Establishing governance bodies to oversee adoption, including a Steering Committee and a Community of Practice
    • Creating a change management strategy, including policies, training, and communications
    • Enabling IPv6 connectivity for Internet-facing websites through a shared service
  • Government of Canada IPv6 Adoption Strategy Overview

July 31: Standard on Web Accessibility

  • Phase 3: All pages must comply with all requirements
  • TBS standard full text requirements

July 31: Standard on Web Usability

  • Phase 1: Appendix C and Appendix D in addition to two new notices for "Accessibility" and "Social media"
  • TBS standard full text requirements

June 30: Standard on Web Interoperability

  • Phase I: Implementation of Web Feed — Atom Syndication Format
  • Web feed requirements are part of Appendix B

June: Web Renewal Initiative

  • Moratorium placed on creation of new public facing websites and purchase of Web content management, analytics, online engagement and search solutions.

April 11: Web Experience Toolkit (WET)

  • WET version 3.1 is released and available from GitHub and support for WET version 2.3 is removed

April 01: Standard on Optimizing Websites and Applications for Mobile Devices

  • Approval of device-based mobile applications by the departmental Senior Delegate Officers (SDO); excluding websites
  • Standard on Mobile Applications Project Overview

April: Web Renewal Initiative

  • The Standard on Optimizing Websites and Applications for Mobile Devices and Social Media Account Management Standard come into effect.

January 31: Effective date of Standard on Privacy and Web Analytics

  • TBS standard full text requirements

Web Renewal Initiative

  • A social media account management service is being procured.
  • A common Google search service will be implemented for the Government of Canada website presence.
  • A government analytics framework will be developed to enable departments and agencies to track and analyze user traffic consistently.

2012

September 17: Web Experience Toolkit (WET)

  • TBS launched version 3.0
  • WET is available from GitHub

July 31: Standard on Web Accessibility

  • Additional Web pages that provide the most important information and services for individuals and businesses including rights and benefits
  • Additional Web pages that are the most frequently used
  • TBS standard full text requirements

July 01: Effective date of Standard on Web Interoperability

  • TBS standard full text requirements

February 29: Standard on Web Accessibility

  • All Web site home pages and pages referenced from Web site home pages
  • Significant number of Web pages that provide the most important information and services for individuals and businesses including rights and benefits
  • Significant number of Web pages that are the most frequently used
  • All new Web pages published post October 1, 2011 must immediately conform
  • TBS standard full text requirements

2011

September 28: Effective date of Standard on Web Usability

  • TBS standard full text requirements

August 01: Effective date of Standard on Web Accessibility

  • TBS standard full text requirements
