Common accessibility issues

Common accessibility problems are listed below according to their WCAG 2.1 Level AA success criterion.

Success Criterion 4.1.1 Parsing

We use W3C Markup Validation Service to validate the HTML. Please use proper HTML to markup the content of the applications. Validate HTML after JavaScript injection.

Success Criterion 3.3.2 Labels or Instructions

Using the Wave Firefox extension to check for accessibility in forms we can ensure that the form elements (fields) have label associated with them.

Success Criterion 1.4.3 Contrast (Minimum) and Success Criterion 1.4.11 Non-text Contrast

Text and non-text (button, icons that have function) need to pass a contrast ratio of 4.5:1. This can be done using the Colour Contrast Analyser to test if the colours have a good ratio for Level AA. Normally we try to make it level AAA if you possible. It's an easy solution to make a site even more accessible

Success Criterion 1.3.2 Meaningful Sequence

Content on pages are using table elements instead of styles (CSS) to layout the content. This becomes an issue of verbosity when a screen reader encounters this content. This also fails Success Criterion 1.4.10 Reflow because the table does not reflow when in mobile or at a smaller screen.

Success Criterion 2.4.6 Headings and Labels

Text that is found on the web should be markup as heading using proper HTML elements for heading (h1, h2, h3, etc...) depending on the hierarchy of the page. Proper descriptive headings.

Success Criterion 2.1.1 Keyboard

All clickable elements should also be accessible via keyboard.

Instances where the use of the title attribute to relay information in a hover state is not visible to users who use keyboard only and many screen readers do not read the value of the title attribute, must also not to forgot about mobile and touch screens users won't see this information either.

Success Criterion 1.1.1 Non-text Content

Missing alt attribute on images. Example is the "multilingual18.gif" in copy form, add the text from title attribute and remove title attribute. Use proper alt text to describe function of linked image.

Images not using proper alt text in the alt attribute to describe the function of the image and not a description of what the image is. Folder image on copy page is using a alt text that has the text "folder", which does not need to be there for decorative images. Please use an empty alt attribute alt="" for all decorative images.

Success Criterion 4.1.2 Name, Role, Value

Using a details (expand / collapse) function in the table which isn't marked up properly. When a screen reader user encounters the function, they are unaware that there is more information related to it. Please refer to Using the WAI-ARIA aria-expanded state to mark expandable and collapsible regions on the W3C website on how to markup this function properly.

Using a tab layout without the proper Roles to define it to a AT device that it's a tab layout. Here's an article on how to build an accessible tap panel.

Success Criterion 2.2.1 Timing Adjustable

If automatic timeout is not set to over 20 hours, you have to give them a way to extend the time.

Success Criterion 2.4.4 Link Purpose (In Context)

Ensure that all links work and none are broken links.
